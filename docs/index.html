Quantum LISP Reasoning Engine

Quantum LISP compiles symbolic reasoning programs into deterministic IR, projects the program into QSO fabric patches and overlaps, runs backend diagnostics, and persists a non-mutating reasoning trace.

(hypothesis hyp.bridge obs.memory)
(entangle obs.memory hyp.bridge :kind dependency :weight 0.7)
(project future.bridge :horizon 3 :using (qiskit pennylane cirq itensor))

Runtime Surfaces

The repository is built for local agents, automation workflows, and public inspection. Every major surface keeps the same bias: deterministic contracts first, optional native engines second.

MCP Tools

Expose QSO create/read/patch, quantum execution, Quantum LISP analysis, transport, identity, and snapshot operations.

Quantum Backends

Qiskit, PennyLane, Cirq, and ITensor adapters run native engines when installed and stable fallbacks otherwise.

QFF Snapshots

Export and import replayable object state for portability, evidence bundles, and handoffs.

Solis Automation

Property, APN, investigation, and graph workflows consume shared QSO state instead of creating parallel ledgers.

Identity State

Identity events, credentials, policy publishing, and verification bundles sit on the same event-sourced substrate.

Visualization

XR, scene projection, cognitive render data, and preview artifacts make state inspectable beyond logs.
